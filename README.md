# Таблица подсчётов

Приложение и MCP-сервер для расчёта интерфейсных оценок и дорожных карт.

## Быстрый старт

Из репозитория:

```bash
npm run start
```

Что делает эта команда:
- если зависимостей ещё нет, ставит их и в корне, и в `mcp-server`
- поднимает приложение
- поднимает локальный MCP-сервер
- открывает приложение в браузере

Если нужно скачать и сразу запустить одной строкой:

```bash
git clone git@github.com:Darlingfxx02/tablica-podschetov.git && cd tablica-podschetov && npm run start
```

## Как работает связка с MCP

`npm run start` всегда запускает приложение вместе с MCP. Это основной сценарий.

Дальше возможны два режима:
- если вы сами открыли приложение через `npm run start`, окно уже готово принимать изменения
- если другая нейросеть поднимет свой MCP-процесс отдельно, он будет работать с тем же state-файлом, и открытое окно подхватит изменения автоматически

То есть таблица, которую генерирует MCP, попадает в уже открытое приложение, а не в какой-то отдельный временный инстанс.

## Подключение в AI-клиент

Для MCP-клиентов важно использовать прямой stdio-entrypoint, а не `npm run ...`, чтобы в протокол не попадал служебный вывод `npm`.

Команда:

```bash
node /ABSOLUTE/PATH/TO/tablica-podschetov/scripts/mcp-stdio.mjs
```

Пример конфигурации:

```json
{
  "mcpServers": {
    "estimate-table": {
      "command": "node",
      "args": [
        "/ABSOLUTE/PATH/TO/tablica-podschetov/scripts/mcp-stdio.mjs"
      ]
    }
  }
}
```

Отдельно запускать MCP вручную обычно не нужно:
- для живой работы пользователя достаточно `npm run start`
- для AI-клиента MCP будет подниматься его собственной средой через `mcp-stdio.mjs`

## MCP Protocol

Основные правила работы через MCP вынесены в [mcp-server/MCP_USAGE_GUIDE.md](/Users/darlingfxx/Мои%20проекты/Работа/Таблица%20подсчетов/mcp-server/MCP_USAGE_GUIDE.md).

Коротко:
- по умолчанию считаем интерфейсный контур, а не full-cycle разработку
- по умолчанию структура оценки должна быть компактной: одна глобальная секция `Проектирование` и одна глобальная секция `Дизайн`
- экраны и модули внутри этих секций должны быть сгруппированы divider-заголовками вроде `Главная`, `Сайдбар`, `Лендинг`
- описания задач должны перечислять конкретные элементы блока по фич-листу, а не давать абстрактное описание раздела
- `custom`-секции нужно избегать
- перед созданием ролей агент обязан уточнить у пользователя состав исполнителей и их зону ответственности
