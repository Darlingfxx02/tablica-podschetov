# Таблица подсчётов

Приложение и MCP-сервер для расчёта интерфейсных оценок и дорожных карт.

## MCP Protocol

Основные правила работы через MCP вынесены в [mcp-server/MCP_USAGE_GUIDE.md](/Users/darlingfxx/Мои%20проекты/Работа/Таблица%20подсчетов/mcp-server/MCP_USAGE_GUIDE.md).

Коротко:
- по умолчанию считаем интерфейсный контур, а не full-cycle разработку
- по умолчанию структура оценки должна быть компактной: одна глобальная секция `Проектирование` и одна глобальная секция `Дизайн`
- экраны и модули внутри этих секций должны быть сгруппированы divider-заголовками вроде `Главная`, `Сайдбар`, `Лендинг`
- описания задач должны перечислять конкретные элементы блока по фич-листу, а не давать абстрактное описание раздела
- `custom`-секции нужно избегать
- перед созданием ролей агент обязан уточнить у пользователя состав исполнителей и их зону ответственности
